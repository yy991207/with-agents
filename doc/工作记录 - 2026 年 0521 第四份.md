## 主要修改点

### 中优 A 后端核心扩展

1. **共享 tool 集合**(`llm/tools.py` + `llm/deep_agents.py`):
   - 实装 3 个业务 tool:`current_time` `http_get` `web_search`
   - `http_get` 含 SSRF 防护(屏蔽 localhost/private IP)+ 100KB 截断 + UA 标识
   - `web_search` 走 DuckDuckGo HTML 接口,无 API key,正则抽取 result
   - reply 模式 deep_agent 注入这 3 个 tool,think 模式仍空 prompt 禁止调用
2. **单 agent retry think**(`core/task_manager.py`):
   - 仅 THINK_DONE 状态允许 retry(主循环阻塞在 _wait_decision 无并发冲突)
   - fire-and-forget 起独立子任务,异常通过 hub 推 think.failed 事件
   - 完成后 publish 一次 task.state THINK_DONE 携带最新 available_agents
   - 路由 KeyError→404 ValueError→409 NotImplementedError→501 完整覆盖

### 中优 B 后端基础设施

3. **session 删除**(`storage/mongo.py` + `routes/sessions.py`):
   - 删除 session 同时级联删除其下所有 rounds
   - 有进行中 round 时拒绝并返回 409
   - DELETE /sessions/{session_id} → 204/404/409
4. **LLM 错误友好化**(`core/errors.py`):
   - `humanize_llm_error` 把 timeout/429/401/403/5xx/context length 等英文错误映射为中文短句
   - 接入 task_manager 三处:think.failed / reply.error / task.unrecoverable
   - 完整堆栈仍记 logger 给运维 用户只看友好提示
5. **配置版本回滚**(`storage/mongo.py` + `routes/agents.py`):
   - 新增 `agent_history` collection 独立存归档
   - upsert_agent 写新值前自动归档当前值
   - GET /api/agents/{name}/history 列版本(降序)
   - POST /api/agents/{name}/revert body {target_version} 回滚到指定版本(新版本号自增)
6. **judge 限频**(`core/rate_limit.py`):
   - 进程内滑动窗口令牌桶 默认 10/60s
   - asyncio.Lock 保护 time.monotonic 防系统时间回拨
   - 接入 routes/decide.py 仅当 choice=auto 时检查 → 429

## 异常处理手法

| 场景 | 处理 |
|---|---|
| LLM 调用各种错误 | humanize_llm_error 中文映射 用户看友好提示 运维查 logger 看堆栈 |
| 改 prompt 失败 | DB 已 upsert 后 reload 异常 自动反向回滚 + 500 错误 |
| revert 失败 | 同 upsert 处理 自动恢复 |
| session 含进行中 round 被删 | 409 拒绝 提示先取消或等完成 |
| judge 被狂点 | 第 11 次 429 + 中文提示"judge 限流 请 N 秒后重试" |
| retry think 在错误状态触发 | 409 提示状态机不允许 |
| http_get 内网 IP | 提前拒绝防 SSRF 不发请求 |
