## 异常处理手法

- 旧 conda 环境 `deepagent` 因 langchain 版本错位无法跑通 deepagents 0.6.x:新建独立 conda 环境 `multi-chat` 隔离 不动旧环境 防止干扰别的项目
- `run.sh` 启动前主动 ping MongoDB 27017 不可达就立即报错并提示用户先 `./run.sh mongo` 不盲启动 uvicorn
- `run.sh` 启动前检查端口 8002 已被占用时不启动 而是提示孤儿进程 PID 让用户手动确认
- `run.sh` 不硬编码 conda 路径 用 `PYTHON_BIN=${PYTHON_BIN:-python}` 让用户通过 `conda activate multi-chat` 后默认 PATH 里的 python 即可工作 也支持环境变量覆盖
