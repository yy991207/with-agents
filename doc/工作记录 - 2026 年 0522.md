## 二、provider profile 全栈集合(本日重头)

### 2.1 改动背景

参考用户提供的截图(类似"配置文件 [claude.n1n] [+] [✎] [删]"的提供商管理 UI),需求是:

- 每个 deepagent **独立**配置 provider(base_url + api_key + 可用模型列表)
- provider 配置存数据库,前端可 CRUD,**不再 yaml 硬编码 key 和 base_url**
- 4 个 agent 可以分别接入不同 provider(全部 GLM,或 2qwen+2deepseek 等任意搭配)

之前架构所有 agent 共享 `settings.key + settings.base_url`,改完后每个 agent 引用一个 profile,profile 集合中央管理。

### 2.2 主要修改点

**新增 collection `provider_profiles`**:
```js
{ name, provider_type, base_url, api_key, models: [{model_id, label}], version, updated_at }
```

**`agents` collection 加字段** `profile_name`,引用 profile.name。

**新增 API**:
- `GET /api/profiles` 列表
- `GET /api/profiles/{name}` 单条
- `POST /api/profiles` 新建(409 重名)
- `PUT /api/profiles/{name}` 增量更新
- `DELETE /api/profiles/{name}` 删除(409 仍被引用)
- `PUT /api/agents/{name}` body 加 profile_name 字段

**API key 安全**:mongo 内明文存(本地 mongo-data 已 .gitignore),接口返回时 mask 成 `首3...末4`,前端永远看不到完整 key。

**deep_agent 工厂改造**:`_build_one` 接收 profile,ChatOpenAI 用 profile.api_key + profile.base_url,不再读 settings 顶层。`registry.initialize/reload` 签名加 profiles_by_name 字典。

**前端 SettingsDrawer 大改**:
- 顶部新加"提供商配置"Card:Select 切换 profile + 新建/编辑(Modal)/删除按钮
- ProfileModal 含 base_url + api_key(Password)+ 多模型行(动态增删)
- API key 编辑模式空字符串 = 不修改
- 4 个 agent tab 内加"API 提供商"下拉 + "模型"下拉(对应 profile.models)
- 模型字段允许"自定义"切换到 Input

### 2.3 异常处理

- **POST 重名**:409
- **DELETE 仍被引用**:409 + 中文 detail "profile X 仍被 N 个 agent 引用 无法删除"
- **PUT 更新失败 reload 异常**:仅日志,因为 PUT 已落库,reload 失败不应 rollback
- **agents PUT 引用不存在的 profile**:400
- **历史数据迁移**:启动时 `update_many({profile_name: {$exists: false}}, $set: 默认)`,把老 agent 补 profile_name 字段——这是修复一个真实 bug(见下)

### 2.4 集成期发现的真实 bug

**现象**:测试中 DELETE "默认" profile 返回 204(应返 409),"默认" profile 被错误删除。

**根因**:老 mongo 中 4 条 agent 中有 3 条(DeepSeek/Kimi/Qwen)**没有 `profile_name` 字段**(只有 GLM 因为我刚才 PUT 时新加的有)。`storage.delete_profile` 用 `count_documents({"profile_name": "默认"})` query 漏算了 3 条没 profile_name 字段的 agent。

**修复**:`seed_from_yaml` 启动时增加迁移逻辑,扫所有 `profile_name $exists: false` 的 agent 文档,$set 为 "默认"。日志可见 `迁移 agents 补齐 profile_name 字段 modified=3`。

### 2.5 涉及模块

后端:`core/models / storage/{base,mongo} / llm/deep_agents / main / routes/{agents,profiles}`
前端:`state/{types,reducer,converters} / api/http / hooks/useSettings / components/SettingsDrawer`
测试:`test_profiles_api 11 用例 / test_storage +7 / test_agents_api +2 / test_deep_agents 改签名 +2`

### 2.6 测试情况

- 后端 pytest **134 passed + 1 skipped**(从 111 → 134,新增 23)
- 前端 type-check 0 错,npm run build 通过
- 真实环境验证:

  - DELETE 默认 profile → 409
  - DELETE 无引用 profile → 204
  - 启动迁移 modified=3 自动补齐老数据

## 三、bug 修复回顾

### 3.1 toolCalls.map of undefined(两轮)

**第一轮**(b4c03ab):切换 session 时崩。
- 根因:后端 `/history/{id}` 返回 mongo 文档原样 snake_case,reply 不含 toolCalls。前端直接当 RoundView[] 用,ReplyBubble 调 `reply.toolCalls.map()` 时 undefined。
- 修复:`useSession.switchSession` 加 `convertRound` 转换。

**第二轮**(6192774):页面刷新时仍崩。
- 根因:抗刷新 bootstrap 流程里 App.tsx 也调 getHistory 但**没**经 convertRound。
- 修复:抽 `state/converters.ts` 共享模块,useSession + App.tsx bootstrap 都 import 同一份。

### 3.2 /sessions sessionId undefined → DELETE 404(88aa362)

后端 GET /sessions 返回 `[{session_id, title, created_at, updated_at}]` snake_case,前端 SessionMeta 期待 `{sessionId, updatedAt}` camelCase。SessionDrawer 渲染时 `item.sessionId === undefined`,点删除 → `DELETE /sessions/undefined` → 404。

修复:`useSession.refreshSessions` 拉到结果后 `list.map(convertSession)` 再 dispatch。

### 3.3 spec §3 架构图错标(3b23d08)

架构图标"OpenAI Chat Client(autogen)",实际后端用 `langchain-openai.ChatOpenAI`。改正描述。

## 四、工程脚本进化:`./run.sh up` 一键启动

### 4.1 改动背景

用户反馈每次启动要做 4 步:`conda activate multi-chat / cd / 起 mongo / ./run.sh start`,期望封装成一句话。

### 4.2 落地

- 顶部 `ensure_conda_env`:自动 source `conda.sh` 然后 `conda activate multi-chat`,已激活时跳过
- `PYTHON_BIN` 默认指向 `$HOME/miniconda3/envs/multi-chat/bin/python`,不依赖外部 activate
- `preflight` 检测 mongo 不可达时自动 `docker compose up -d mongo`,等 15s 仍不可达才退出
- 新增 `up` 子命令:web/dist 缺失自动 build → mongo 自动起 → uvicorn 后台
- 修复 `cmd_start` 一个 bug:用 subshell 写 PID 文件丢失,改成显式 cd

### 4.3 用法

```bash
/Users/yang/multi-chat/run.sh up         # 任意目录任意 shell 一句话
```
